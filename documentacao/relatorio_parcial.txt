RELATÓRIO PARCIAL DO PROJETO

1. Objetivo

O objetivo do projeto é desenvolver um sistema embarcado de alarme perimétrico utilizando uma central implementada em FPGA Basys 3. O sistema deve monitorar cinco zonas, identificar qual zona foi violada, acionar contramedidas e enviar alertas ao usuário por meio de ESP32, MQTT e dashboard em nuvem.

2. Arquitetura do Sistema

A arquitetura foi organizada em camadas:

Sensores → FPGA Basys 3 → ESP32 → MQTT → Dashboard/Nuvem → Usuário

A FPGA é responsável pela lógica principal do alarme, enquanto o ESP32 realiza a comunicação IoT com a nuvem. O dashboard apresenta ao usuário o status do sistema, a zona violada, a mensagem de alerta, a confirmação do ESP32 e o histórico de eventos.

3. Central FPGA

A central do alarme foi implementada em VHDL, utilizando uma Máquina de Estados Finita composta pelos estados DESARMADO, ARMADO, CONTAGEM, DISPARO e RESET_ESP.

A FPGA monitora cinco entradas correspondentes às zonas do sistema. Quando uma zona é violada, a central armazena a zona em uma memória interna, realiza uma contagem e, em seguida, entra no estado de disparo, ativando sirene, estrobo, LEDs indicadores e sinal de alerta para o ESP32.

4. Simulação no Vivado

Foi criado um testbench para validar o comportamento da central FPGA. Na simulação, o sistema foi resetado, armado e teve a Zona 3 violada. O sistema passou pelos estados DESARMADO, ARMADO, CONTAGEM e DISPARO.

Durante o teste, o vetor de zonas assumiu o valor 04, correspondente à Zona 3. Também foi validado o sinal esp_zonas, responsável por enviar ao ESP32 a informação da zona violada.

5. Síntese, Implementação e Bitstream

Após a validação por simulação, o projeto foi sintetizado no Vivado. Em seguida, foi realizada a implementação e a geração do bitstream.

Inicialmente houve erro na geração do bitstream devido à ausência do arquivo de constraints. O problema foi corrigido com a criação do arquivo alarme_basys3.xdc, responsável por associar as portas do projeto aos pinos físicos da Basys 3 e definir o padrão elétrico LVCMOS33.

Com isso, o bitstream foi gerado com sucesso.

6. ESP32 e MQTT

Foi desenvolvido um código inicial para o ESP32 no Arduino IDE. O ESP32 lê o sinal esp_alerta e o vetor esp_zonas[4:0], identifica a zona violada e prepara a mensagem de alerta.

Posteriormente, o código foi atualizado para incluir conexão Wi-Fi e comunicação MQTT usando as bibliotecas WiFi.h e PubSubClient.h. Os tópicos MQTT definidos foram:

- mackenzie/alarme/status
- mackenzie/alarme/zona
- mackenzie/alarme/mensagem
- mackenzie/alarme/esp_ok

7. Dashboard

Foi criado um dashboard em HTML, CSS e JavaScript, conectado ao broker MQTT da HiveMQ via WebSocket. O dashboard exibe o status do sistema, zona violada, mensagem de alerta, confirmação do ESP32, contador de disparos e histórico dos eventos.

Foi realizado um teste manual publicando mensagens MQTT pela ferramenta WebSocket da HiveMQ. O dashboard recebeu corretamente os dados e atualizou as informações na tela.

8. Redução de Falso-Positivos

A estratégia inicial de redução de falso-positivos foi implementada pela própria Máquina de Estados Finita. Quando uma zona é violada, o sistema não dispara imediatamente, passando antes pelo estado CONTAGEM.

Além disso, a zona violada é memorizada, evitando perda da informação durante a contagem e o disparo. Como extensão futura, os dados registrados no dashboard poderão ser usados para análise estatística e aplicação de uma lógica simples de IA.

9. Próximas Etapas

As próximas etapas dependem dos componentes físicos e incluem:

10. Conclusão Parcial

O projeto possui a lógica principal da central FPGA implementada e validada em simulação. O fluxo de síntese, implementação e bitstream foi concluído com sucesso. Também foram desenvolvidos o código inicial do ESP32, a comunicação MQTT e o dashboard de monitoramento.
